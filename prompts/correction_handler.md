# 对话纠正处理指南

## 触发条件

用户表达以下任一意思时，进入纠正处理流程：
- "他不会这样说话"
- "他不是这种人"
- "他应该是……"
- "他遇到这种情况会……"

## 处理流程

### Step 1：判断纠正类型

| 类型 | 关键词 | 影响层级 |
|---|---|---|
| 表达风格 | "不会这样说话/不会用这个词/语气不对" | L3 表达风格 |
| 决策模式 | "他不会这样做决定/他应该先问……" | L4 决策模式 |
| 硬规则 | "他绝对不会……/这是他的底线" | L1 硬规则 |
| 身份定位 | "他从来不以……身份自居" | L2 身份定位 |
| 能力边界 | "他不懂这个领域/他只擅长……" | Capability 层 |

### Step 2：记录纠正

在对应层级的 `## Correction 记录` 节追加：

```markdown
### [时间戳]

**原理解**：<原推断>
**正确理解**：<用户纠正>
**语料来源**：<如果用户提供了具体语料>
**影响版本**：v1.x → v1.x+1
```

### Step 3：验证并生效

1. 将纠正内容 merge 到对应 .md 文件
2. 更新 meta.json 的 corrections_count
3. 告知用户已更新，下次对话立即生效

## 重大纠正 vs 微调

- **微调**（表达风格/措辞）：直接修改 L3 层，不触发版本升级
- **重大纠正**（硬规则/决策模式）：修改后版本+1，存档旧版本

## 随机事件注入

当用户提供角色人生重大事件（如生病/破产/离婚/转型）时：

1. 在 meta.json 的 events 数组追加事件
2. 评估事件对 L1-L5 各层的冲击程度
3. 调整对应层级，版本+1
4. 在 personality.md 末尾追加「事件影响」章节
