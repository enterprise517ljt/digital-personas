# Digital Persona 建立标准化流程

> 版本：v1.0.0 | 日期：2026-04-09

---

## 一、采集目标（量化标准）

| 内容类型 | 最低数量 | 目标数量 | 来源 |
|---------|---------|---------|------|
| 视频标题 | 100条 | **200条+** | 抖音/B站账号主页 |
| AI字幕/文案 | 50条 | **100条+** | 视频字幕或视频文本 |
| 高赞评论 | 30条 | **50条+** | 视频评论区 |
| 口头禅 | **30条** | 50条+ | 从字幕+评论提取 |
| 账号基础信息 | 完整 | 完整 | 主页/搜索结果 |

**总语料目标：200条以上**

---

## 二、采集流程

### 阶段 A：账号发现与验证

```
1. 确定平台（抖音/小红书/B站/微信视频号）
2. 确定账号名称
3. 通过搜索确认账号存在
4. 验证：粉丝数 / 获赞数 / 签名
```

**搜索验证命令：**
```bash
python3 scripts/fetch_profile.py <账号关键词> <平台>
```

---

### 阶段 B：六类内容采集

| # | 内容类型 | 采集方法 | 文件输出 |
|---|---------|---------|---------|
| 1 | 账号基础信息 | 主页文本提取 | `corpus/00_profile.md` |
| 2 | 视频标题（>200条） | 主页列表滚动抓取 | `corpus/01_titles.md` |
| 3 | 视频AI字幕/文案 | 视频页抓取 + AI摘要 | `corpus/02_captions.md` |
| 4 | 高赞评论（>30条） | 评论区抓取 | `corpus/03_comments.md` |
| 5 | 话题/标签 | 主页标签 + 视频hashtag聚合 | `corpus/04_tags.md` |
| 6 | 直播连麦语料 | 直播回放评论提取 | `corpus/05_live.md` |

---

### 阶段 C：采集工具链

**CDP 连接（Chrome 开启调试模式）：**
```bash
open -a "Google Chrome" --args --remote-debugging-port=28800
```

**标准采集脚本：**
```bash
# 1. 采集账号主页（账号信息 + 视频标题）
python3 scripts/fetch_profile.py <账号关键词>

# 2. 批量采集视频字幕
python3 scripts/fetch_captions.py <角色名> <视频URL列表>

# 3. 采集评论区
python3 scripts/fetch_comments.py <角色名>
```

**备用：page_snapshot.py（无需 WebSocket）：**
```bash
TAB_ID=$(curl -s -X PUT http://localhost:28800/json/new | python3 -c "import json,sys; print(json.load(sys.stdin)['id'])")
python3 ~/.qclaw/browsers/.../page_snapshot.py --target $TAB_ID --cdp-url http://localhost:28800 text > corpus/01_titles.md
```

## 三、分析与建模流程

### 阶段 D：语料预处理

```
1. 合并 corpus/*.md 所有文件 → raw_corpus.txt
2. 去重、过滤垃圾字符
3. 按内容类型分类标注
```

### 阶段 E：L1-L5 角色建模

从语料中提取以下维度：

| L层 | 提取来源 | 关键指标 |
|-----|---------|---------|
| L1 三观 | 字幕+评论 | 世界观、人生观、价值观 |
| L2 身份 | 账号简介+评论 | 身份认同、立场、性格 |
| L3 认知 | 内容主题 | 判断框架、核心观点 |
| L4 表达 | 字幕+评论 | 语气、句式、**30+口头禅** |
| L5 行为 | 视频结构+评论 | 内容模式、互动方式 |

### 口头禅提取标准

从原始语料中筛选：
1. 重复出现 ≥3 次的短语
2. 语气词 + 实词组合（如"买个der"、"认清高低位"）
3. 金句/段子（传播度高的句子）
4. 反问句/感叹句模板

**最低 30 条，目标 50 条**

## 四、文件输出标准

每个角色的标准目录结构：

```
personas/<name>/
├── meta.json              # 角色元数据（平台/账号/粉丝/赛道/签名语录）
├── layers/
│   ├── l1_foundations.md  # L1 三观层
│   ├── l2_foundations.md  # L2 身份层（含口头禅）
│   ├── l3_foundations.md  # L3 认知层
│   ├── l4_foundations.md  # L4 表达层
│   └── l5_foundations.md  # L5 行为层
└── corpus/                # 原始语料
    ├── 00_profile.md      # 账号基础信息
    ├── 01_titles.md       # 视频标题（>200条）
    ├── 02_captions.md     # 字幕/文案（>100条）
    ├── 03_comments.md     # 高赞评论（>30条）
    ├── 04_tags.md         # 话题/标签
    └── 05_live.md         # 直播语料
```

## 五、SKILL.md 路由标准

新增角色时，在 SKILL.md 更新：

```
| 优先级 | 关键词 | 角色 | 赛道 |
|--------|--------|------|------|
| N      | 关键词1/关键词2/... | <name> | <赛道> |
```

关键词规则：
- 覆盖主赛道（核心词）
- 覆盖受众痛点词
- 覆盖内容主题词
- 3-5个词，用 `/` 分隔

## 六、采集数量达标检查清单

在完成角色前，必须确认：

- [ ] 账号信息完整（粉丝/获赞/签名/认证）
- [ ] 视频标题 ≥ 100 条（目标 200 条）
- [ ] 字幕/文案 ≥ 50 条（目标 100 条）
- [ ] 高赞评论 ≥ 30 条（目标 50 条）
- [ ] 口头禅 ≥ 30 条（目标 50 条）
- [ ] L1-L5 五个层级完整
- [ ] meta.json 包含 signature_phrases ≥ 30 条
- [ ] SKILL.md 路由已更新
- [ ] GitHub 已推送

**任一语料类指标未达标时，角色版本标记为 "⚠️ 待补采"**
